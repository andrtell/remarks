__The easymath example library__

`./easymath.h`

```c
#ifndef EASYMATH_H
#define EASYMATH_H

int easymath_add(int a, int b);

#endif
```

`./easymath.c`

```c
int easymath_add(int a, int b) {
	return a + b;
}
```

_Given these two files we can compile a shared object file._

```
> cc -fPIC -shared -o libeasymath.so easymath.c
```

__A simple program using the easymath library__

`./program.c`

```c
#include <stdio.h>
#include <easymath.h>

int main(void) {
  int v = easymath_add(1, 2);
  printf("%d + %d = %d\n", 1, 2, v);
}
```

_If we try to compile this program without any extra compiler flags, we will get an error._

```
> cc program.c -o program

program.c:2:10: fatal error: easymath.h: No such file or directory
    2 | #include <easymath.h>
      |          ^~~~~~~~~~~~
compilation terminated.
```

_We can fix this error by using the `-I` flag to specifiy the location of the `easymath.h` header file._

```
> export LIB_DIR=$(pwd)

> cc -I $LIB_DIR program.c -o program

/usr/bin/ld: /tmp/ccCtxbLO.o: in function `main':
program.c:(.text+0x13): undefined reference to `easymath_add'
collect2: error: ld returned 1 exit status
```

_We fixed the first error, but got another error. This time it is the linker that complains it can't find the `easymath_add` function referenced in `main`._

_At this stage the linker needs to know which library provides the function `easymath_add`, so it can record the dependency in the final executable._

_We can fix this error by using the `-L` flag to tell the compiler where to find the `libeasymath.so` shared object file and the `-l` flag to tell the compiler to we wish to link our executable `program` with the `libeasymath.so` shared object file._

```
> export LIB_DIR=$(pwd)

> cc -I $LIB_DIR \
   	 -L $LIB_DIR \
	 program.c \
	 -leasymath \            # -leasymath -> lib + easymath + .so
	 -o program
```
